## I18N-002 — Create i18n.ts initialisation module

**Owner:** frontend
**Depends on:** I18N-001
**Description:** Create `mobile/src/i18n/i18n.ts` following the exact structure defined in `docs/i18n-architecture.md` §4. The file performs a synchronous side-effect init of i18next with `initReactI18next`, imports `tr.json` and `en.json`, detects the system locale via `Intl.DateTimeFormat`, and exports the configured `i18n` instance. Translation JSON files must exist before this file can be written, so this ticket depends on I18N-003.

**Files:**

## I18N-003 — Create tr.json and en.json translation files

**Owner:** frontend
**Depends on:** none
**Description:** Create the two locale files under `mobile/src/i18n/locales/`. `tr.json` is the source-of-truth (kaynak dil) and must contain every key listed in the key inventory in `docs/i18n-spec.md` with Turkish values as shown in `docs/i18n-architecture.md` §8. `en.json` must contain the identical key structure with English values. No key may be present in one file and absent in the other.

**Files:**
- CREATE `mobile/src/i18n/locales/tr.json`
- CREATE `mobile/src/i18n/locales/en.json`
- DO NOT modify any screen files

## I18N-006 — Build SettingsScreen

**Owner:** frontend
**Depends on:** I18N-002, I18N-003, I18N-005
**Description:** Create `mobile/src/screens/SettingsScreen.tsx`. The screen renders a language selection list with two rows ("Türkçe" / "English"). The active language shows a checkmark on the right; tapping a row calls `languageService.setLanguage(lang)` — no confirmation dialog, no local state needed. The header title is set via `useLayoutEffect` + `navigation.setOptions({ title: t('settings.screenTitle') })` so it updates on language change. Visual style must match ProfileScreen: same gradient background and card structure. See the mockup in `docs/i18n-spec.md` and the component skeleton in `docs/i18n-architecture.md` §10.3.

**Files:**
- CREATE `mobile/src/screens/SettingsScreen.tsx`
- DO NOT modify navigation files (that is I18N-007)

## I18N-019 — Manual QA: language switching and persistence

**Owner:** tester
**Depends on:** I18N-006, I18N-007, I18N-013, I18N-018
**Description:** Launch the app on a simulator and manually verify all acceptance criteria from `docs/i18n-spec.md`. Walk through the full scenario matrix below.

Scenarios to cover:
1. Open app for the first time (no AsyncStorage entry) — confirm system locale is detected and applied.
2. Navigate to Profile > Settings — confirm SettingsScreen loads and shows two language rows.
3. Select "English" — confirm all visible text on SettingsScreen, ProfileScreen, and TodoListScreen switches to English immediately without restart.
4. Close and reopen the app — confirm English is still active (AsyncStorage persistence).
5. Switch back to "Türkçe" — confirm all screens revert to Turkish immediately.
6. Open TodoFormScreen — confirm all labels, placeholders, priority options, and reminder options are in the active language.
7. Open TaskDetailScreen — confirm all action labels and banners are in the active language.
8. Open LoginScreen and RegisterScreen — confirm all strings are in the active language.
9. Confirm no screen shows any hard-coded Turkish string while English is active.
10. Confirm `npx tsc --noEmit` exits with zero errors (delegate to I18N-018 result).

**Files:**
- No files modified — record pass/fail results in a QA comment on this ticket
