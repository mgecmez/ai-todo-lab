# Release Notes — v1.0.0 Sprint 2 (BL-016 + BL-034)

**Release Date:** 2026-03-21
**Type:** Security + Performance
**Scope:** Backend (BL-016) + Mobile Frontend (BL-034)

---

## Overview

Sprint 2 delivers two independent improvements: brute-force protection on authentication endpoints (BL-016) and React Native rendering performance optimizations for large todo lists (BL-034).

---

## What's New

### Rate Limiting on Auth Endpoints (BL-016)

- **Login protection:** `/api/auth/login` is limited to **5 requests per 15 minutes** per IP address
- **Register protection:** `/api/auth/register` is limited to **10 requests per hour** per IP address
- **429 response:** Exceeding the limit returns `429 Too Many Requests` with a `Retry-After` header (seconds until window resets)
- **Configuration-driven:** Limits are defined in `appsettings.json` under the `RateLimit` section — no code change needed to adjust thresholds
- **Standard error format:** `{ "status": 429, "message": "Çok fazla deneme yapıldı. Lütfen daha sonra tekrar deneyin." }`

### FlatList Performance Optimization (BL-034)

- **`React.memo` on `TodoItem`:** The list item component now skips re-renders when its props have not changed, preventing unnecessary work when unrelated state updates occur
- **`useCallback` on handlers:** `handleToggle`, `handlePin`, `handleDeleteConfirm`, and `handleDeletePress` are stabilized, ensuring `TodoItem` receives the same function references across renders
- **FlatList render tuning:** `removeClippedSubviews`, `initialNumToRender`, `maxToRenderPerBatch`, and `windowSize` configured for smooth scrolling with 100+ items
