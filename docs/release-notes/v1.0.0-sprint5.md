# Release Notes — v1.0.0 Sprint 5 (BL-033)

**Release Date:** 2026-03-22
**Type:** Infrastructure / Build Pipeline
**Scope:** Mobile

---

## Overview

Sprint 5 migrates the mobile app from Expo Go to EAS Build (Expo Application Services), enabling real native binary production for iOS and Android. The project can now be built with three profiles — development (dev-client), preview (internal distribution), and production (store) — and is fully prepared for App Store / Play Store submission in a future sprint.

---

## What's New

### EAS Build Configuration (BL-033)

- **`mobile/eas.json`** — Three build profiles:
  - `development`: `expo-dev-client`, internal distribution, iOS simulator, Android APK
  - `preview`: Internal distribution APK
  - `production`: Store distribution, Android AAB (App Bundle)
- **`expo-dev-client`** installed — Enables native module support beyond Expo Go limitations
- **`runtimeVersion: { policy: "appVersion" }`** — OTA update runtime versioning tied to `app.json` version
- **EAS Update ready** — `updates.url` configured for EAS Update OTA delivery
- **`docs/eas-build-guide.md`** — Step-by-step developer guide for building and distributing the app

## Developer Setup

First-time EAS setup requires:

```bash
eas login           # Expo account credentials
cd mobile
eas init            # Links project, prints projectId
```

Replace `PLACEHOLDER_PROJECT_ID` in `mobile/app.json` with the printed `projectId`. See `docs/eas-build-guide.md` for the full workflow.

## Known Limitations

- `extra.eas.projectId` in `app.json` is set to `PLACEHOLDER_PROJECT_ID` — each developer must run `eas init` to get a real project ID
- Apple Developer account / certificates not configured — iOS store submission deferred to a future sprint
- No GitHub Actions CI/CD integration — cloud builds triggered manually via `eas build` CLI
