# Release Notes — v0.9.0

**Release Date:** 2026-03-21
**Type:** Feature Release
**Scope:** Mobile (Frontend only)

---

## Overview

v0.9.0 replaces the free-text due date input in `TodoFormScreen` with a native platform date/time picker. Users can now select both date and time via a two-step native dialog on both iOS and Android. A shared `formatDate` utility was also extracted to eliminate code duplication across screens.

---

## What's New

### Native DateTime Picker (`DateTimePickerField`)

- Replaced the manual `TextInput` (which required typing `YYYY-MM-DD`) with a new reusable `DateTimePickerField` component.
- **iOS:** Two-step picker flow — date picker first, then time picker.
- **Android:** Two-step modal dialog flow — date modal, then time modal.
- Dismissing at any step cancels the selection; the existing value is preserved.
- A **Clear** button appears when a date is selected, allowing one-tap removal.
- When due date is cleared, the reminder offset is automatically reset to `null` and the reminder selector is disabled.
- The picker trigger is disabled while the form is saving (`disabled` prop).

### Due Date Pre-population in Edit Mode

- When editing a task that already has a `dueDate`, the picker trigger shows the existing value pre-loaded.
- The picker opens at the current due date/time instead of the system default.

### Time Display in Date Fields

- `TodoListScreen` and `TaskDetailScreen` now display time alongside the date when a due time is set:
  - Time is present → `DD.MM.YYYY HH:mm`
  - Midnight UTC (no time set) → `DD.MM.YYYY`
- Time is rendered in the device's local timezone.

## Technical Changes

### New Files

| File | Description |
|------|-------------|
| `mobile/src/components/DateTimePickerField.tsx` | Reusable date/time picker component encapsulating iOS/Android platform logic |
| `mobile/src/utils/formatDate.ts` | Shared date formatter utility (`formatDate(iso: string): string`) |

### Modified Files

| File | Change |
|------|--------|
| `mobile/src/screens/TodoFormScreen.tsx` | Removed `dueDateText` state and manual `TextInput`; `dueDate` state type changed from `string \| null` to `Date \| null`; integrated `DateTimePickerField` |
| `mobile/src/screens/TodoListScreen.tsx` | Removed inline `formatDate` function; imports from shared utility |
| `mobile/src/screens/TaskDetailScreen.tsx` | Removed inline `formatDate` function; imports from shared utility |
| `mobile/package.json` | Added `@react-native-community/datetimepicker` dependency |

### Dependency Added

```
@react-native-community/datetimepicker
```

Installed via `npx expo install` for Expo SDK compatibility.

## Breaking Changes

None. The backend API contract (`dueDate: string | null`, ISO 8601 UTC) is unchanged.

---

## Bug Fixes

None.

## Out of Scope

| Item | Reason |
|------|--------|
| Backend API changes | `dueDate: DateTime?` already supported ISO 8601 UTC |
| Timezone selector | Standard `Date` API + `toISOString()` is sufficient |
| "All day" mode | Planned for a future sprint |
| Past date restriction | Separate feature |
| Web platform support | iOS/Android priority for Expo managed workflow |

---

## Migration Notes

No action required. The `dueDate` field stored in existing records remains valid — ISO 8601 strings are parsed directly into `Date` objects on the client.
