# Release Notes — v0.10.0

**Release Date:** 2026-03-21
**Type:** Feature Release
**Scope:** Mobile (Frontend only)

---

## Overview

v0.10.0 is a quick-wins sprint that closes five small, self-contained gaps in the mobile app. No backend API changes. All items are additive or refactor-only with no breaking changes.

---

## What's New

### All-Day Mode (BL-001)

- A new **All Day** toggle appears in `TodoFormScreen` when a due date is selected.
- When active, the picker stops after the date step — the time step is not shown on either iOS or Android.
- The saved `dueDate` is midnight UTC (`T00:00:00.000Z`) for the selected calendar day.
- Task list and detail screens display only `DD.MM.YYYY` for all-day tasks (no time component). This uses the existing `formatDate` rule — no display changes required.
- Clearing `dueDate` resets the toggle to `false` automatically.

### Past-Date Warning (BL-002)

- After the picker closes with a confirmed date, if the selected date/time is in the past, an informational `Alert` is shown immediately.
- The warning is soft — the user can dismiss it and still save the task with the past date.
- The alert does not fire when opening an existing task that already had a past due date (no spurious warnings on edit).
- Compatible with All-Day mode: selecting a past calendar day in all-day mode also triggers the alert.

### Toggle → Notification Auto-Cancel / Reschedule (BL-003)

- Marking a task as complete (`isCompleted = true`) automatically cancels its pending local notification.
- Unchecking a completed task (`isCompleted = false`) reschedules the notification if both `dueDate` and `reminderOffset` are still set and the calculated fire time is in the future.
- Logic is in `useToggleTodo.onSuccess` — runs only after a confirmed server response. Optimistic rollback (`onError`) does not affect notifications.
- Safe on Expo Go Android — `notificationService` error guards are already in place.

## Improvements

### `DateTimePickerField` — Placeholder Prop (BL-004)

- Added optional `placeholder?: string` prop (default: `"Select date"`).
- Replaces the previously hard-coded Turkish placeholder string.
- `TodoFormScreen` passes `placeholder="Tarih seçilmedi"` — visible UI is unchanged from v0.9.0.
- Makes the component reusable across different screens and locales.

---

## Refactoring

### `isOverdue` Utility Extraction (BL-005)

- Moved the `isOverdue(dueDate, isCompleted)` helper from both `TodoListScreen` and `TaskDetailScreen` into a shared `mobile/src/utils/isOverdue.ts` module.
- Follows the same pattern established by `formatDate.ts` in v0.9.0.
- No behavioral change — both screens import from the same source.
