# Release Notes — v1.0.0 Sprint 1 (BL-044)

**Release Date:** 2026-03-21
**Type:** Feature Release
**Scope:** Mobile (Frontend only)

---

## Overview

Sprint 1 of the v1.0.0 MVP release adds full multi-language support (i18n) to the mobile app using `react-i18next`. Turkish and English are now supported. The user can switch languages from the new Settings screen; the selection persists across app restarts via AsyncStorage.

This sprint establishes the i18n infrastructure that all future sprints must follow — hard-coded UI strings in source files are no longer permitted.

---

## What's New

### Multi-Language Support (BL-044)

- **Two languages supported:** Turkish (`tr`) and English (`en`)
- **Automatic default:** System locale detection via `Intl.DateTimeFormat`; falls back to English if the system locale is not Turkish
- **Instant switching:** Changing the language in Settings updates all visible screens immediately without restarting the app
- **Persistent preference:** Selected language is saved to AsyncStorage (`@app:language`) and restored on next launch

### New Settings Screen

- Accessible from Profile → Settings
- Displays two language options (Türkçe / English) with a checkmark on the active selection
- Consistent visual style with ProfileScreen (gradient background, card layout)

## Breaking Changes

None. Backend API contract is unchanged.

---

## Rules Going Forward

All new UI strings must be added to `tr.json` first, then `en.json`. Hard-coding Turkish or English strings directly in `.tsx`/`.ts` files is no longer permitted. Any missing translation key will cause a TypeScript compile error.
