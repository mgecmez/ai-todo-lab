# Release Notes — v1.0.0 Sprint 6 (BL-032)

**Release Date:** 2026-03-23
**Type:** Infrastructure / Observability
**Scope:** Mobile

---

## Overview

Sprint 6 integrates Sentry error monitoring into the mobile app. Unhandled exceptions,
native crashes, auth failures, and offline mutation errors are now automatically captured
and reported to the Sentry dashboard for preview and production builds. The development
profile keeps Sentry disabled to avoid noise during local development.

---

## What's New

### Sentry Error Monitoring (BL-032)

- **Automatic capture** — Unhandled JS exceptions, unhandled promise rejections, and
  native crashes are reported automatically in preview and production builds
- **Auth error capture** — Login, register, and refresh token failures are reported via
  `captureAuthError()` with operation type as a tag (`auth.operation`)
- **Offline sync error capture** — Global `MutationCache` `onError` hook reports all
  TanStack Query mutation failures with `mutationKey` and `failureCount` as context
- **User context** — `Sentry.setUser({ id, email })` is set after login and session
  restore; cleared on logout — enables per-user filtering in the Sentry dashboard
- **Source map upload** — `@sentry/react-native/expo` plugin auto-uploads source maps
  during EAS preview and production builds for human-readable stack traces
- **Development profile** — Sentry is initialized with `enabled: false` so local
  development generates no Sentry events

## Setup Required

Before Sentry reporting is active in a build, EAS secrets must be configured:

```bash
eas secret:create --scope project --name SENTRY_DSN --value "<your-dsn>"
eas secret:create --scope project --name SENTRY_AUTH_TOKEN --value "<your-auth-token>"
eas secret:create --scope project --name SENTRY_ORG --value "<your-org-slug>"
eas secret:create --scope project --name SENTRY_PROJECT --value "ai-todo-lab"
```

See `docs/sentry-guide.md` for the full setup walkthrough.

## Known Limitations

- Native crash capture requires a real EAS native build — not available in Expo Go
- Sentry Performance Monitoring (transactions, spans) is not enabled — deferred to a future sprint
- Backend (.NET API) error reporting is not included — frontend only
